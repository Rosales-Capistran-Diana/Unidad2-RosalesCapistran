Diferencias entre 20px, 2em, 3rem

  • 20 px

    • - Su valor es fijo y no cambia en función de otros elementos o configuraciones del usuario.

    - Normalmente se utiliza para elementos gráficos, espaciados pequeños o cuando se necesita un control muy preciso sobre el tamaño.

    - No es muy flexible para diseños responsivos, ya que el tamaño no se adapta a diferentes resoluciones de pantalla.

      Ventajas:

    • Precisión: Ofrece un control muy preciso sobre el tamaño y el espaciado de los elementos.
    • Sencillez: Es fácil de entender y utilizar, ya que representa un punto físico en la pantalla.
    • Perfecto para elementos gráficos: Ideal para imágenes, iconos y otros elementos que requieran un tamaño fijo.


  • 2em

    • - Su valor depende del tamaño de fuente del elemento padre.

    - Útil para crear jerarquías tipográficas y mantener relaciones proporcionales entre los tamaños de fuente dentro de un mismo elemento.

    - Puede ser difícil de gestionar en diseños complejos, ya que los cambios en el tamaño de fuente de un elemento pueden afectar a todos sus elementos hijos.

    - Un pequeño cambio en el tamaño de fuente de un elemento padre puede causar cambios en cascada en todos sus hijos.

      Ventajas:

    • Flexibilidad: Permite crear diseños escalables, donde los elementos se ajustan proporcionalmente al cambiar el tamaño de fuente del elemento padre.
    • Jerarquías tipográficas: Facilita la creación de jerarquías visuales claras al relacionar el tamaño de los elementos con el tamaño de la fuente del texto.
    • Adaptabilidad: Se adapta bien a diferentes tamaños de pantalla y preferencias del usuario.


  • 3rem

    • - Similar a "em", pero su valor depende del tamaño de fuente del elemento raíz

    - Recomendada para establecer tamaños de fuente base y crear sistemas de diseño responsivos y accesibles.

    - Calcular tamaños exactos en rem puede volverse complejo, especialmente cuando se anidan varios elementos y se aplican diferentes factores de escala.

    - Es una herramienta poderosa para crear diseños escalables y responsivos, pero no es perfecta. Al comprender sus limitaciones y utilizarla de manera adecuada, puedes aprovechar al máximo sus ventajas y evitar problemas comunes.

      Ventajas:

    • Escalabilidad global: Permite crear sistemas de diseño escalables y consistentes en toda la página.
    • Independencia del contexto: El tamaño de los elementos se define en relación al tamaño de fuente del elemento raíz, lo que evita problemas de anidamiento y hace que los cálculos sean más sencillos.
    • Accesibilidad: Facilita la creación de sitios web accesibles, ya que permite a los usuarios ajustar el tamaño de fuente de forma global sin afectar el diseño.


     Otras unidades comunes
    Unidad Descripción corta
    % (porcentaje) Expresa un tamaño como un porcentaje del tamaño del contenedor padre. Es muy útil para crear diseños flexibles y responsivos.
    vw (Viewport width) Representa un porcentaje del ancho de la ventana del navegador. Ideal para crear diseños que se ajusten al tamaño de la pantalla.
    vh (Viewport height) Similar a vw, pero representa un porcentaje de la altura de la ventana del navegador.
    vmin Toma el valor más pequeño entre vw y vh.
    vmax Toma el valor más grande entre vw y vh.
    ex Equivale a la altura de la letra "x" en la fuente actual.
